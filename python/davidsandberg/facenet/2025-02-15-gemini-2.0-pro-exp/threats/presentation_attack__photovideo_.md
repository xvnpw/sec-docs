## Deep Analysis: Presentation Attack (Photo/Video) on Facenet-based Application

### 2. Scope

### 4. Deep Analysis of the Threat

#### 4.1.  `facenet` Vulnerability

The `facenet` library, at its core, is designed for face *recognition*, not *liveness detection*.  It focuses on:

*   **Face Detection:**  Locating faces within an image.
*   **Feature Extraction:**  Generating a numerical representation (embedding) of the face's unique features.
*   **Similarity Comparison:**  Calculating the distance between embeddings to determine if two faces match.

`facenet` does *not* include any mechanisms to distinguish between a live person and a photograph, video, or other representation of a face.  It treats all input images equally, assuming they represent a real, present face. This is the fundamental vulnerability that enables presentation attacks.

#### 4.2. Attack Vector Enumeration

An attacker can exploit this vulnerability using various methods, including:

*   **High-Resolution Printouts:**  Printing a high-quality photograph of the target user's face.  This is the simplest and most readily available attack.
*   **Digital Photographs on a Screen:**  Displaying a photograph of the target user on a smartphone, tablet, or laptop screen.  This allows for dynamic adjustments (brightness, angle) to improve the chances of success.
*   **Video Replay Attacks:**  Playing a video recording of the target user on a screen.  This can be more convincing than a static image, as it includes movement.
*   **3D Masks:**  Using a realistic 3D mask of the target user's face.  This is a more sophisticated attack, but masks are becoming increasingly accessible.
*   **Deepfakes (Advanced):**  Creating a synthetic video of the target user using deep learning techniques.  This is the most advanced and potentially most difficult attack to detect.
* **Projector Attack:** Projecting image of target user face on the wall or other surface.
